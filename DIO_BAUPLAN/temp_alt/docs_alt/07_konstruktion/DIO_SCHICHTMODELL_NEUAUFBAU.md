# DIO Schichtmodell für den Bauplan

Dieses Dokument ist eine Referenz fuer den Bauplan des DIO-Kerns. Ziel ist
eine saubere Trennung zwischen Realitaet, Wahrnehmung, Innenwirkung, Gedanke,
Handlung und Konsequenz.

## 1. Welt

Chartdaten kommen als reale Aussenwelt in DIO:

- OHLCV
- Zeit
- Bewegung
- Preisverlauf

Die Welt deutet nicht. Sie ist der reale Ablauf.

## 2. Sehen

DIO bildet aus der Welt eine sichtbare Formspur:

- Form
- Richtung
- Verdichtung
- Bereich
- Wiederkehr
- zeitliche Tiefe

Sehen ist noch keine Handlung.

## 3. MCM-Feld

Die gesehene Form erzeugt eine innere Wirkung:

- Spannung
- Tragfaehigkeit
- Druck
- Ruhe
- Nachhall
- Kohärenz oder Bruch

Das MCM-Feld ist die innere Resonanz auf die gesehene Welt. Es ist kein
Trading-Signal, sondern ein Wahrnehmungs- und Spannungsraum.

## 4. Memory

DIO fragt:

`Kenne ich diese reale Form/MCM-Lage oder eine verwandte Form aus Erfahrung?`

Memory ankert an realer Form und realer MCM-Wirkung. Es darf nicht an einer
freien Hypothese als eigener Realitaet ankern.

## 5. Thought

DIO bildet aus Sehen, MCM und Memory eine Hypothese:

`Vielleicht laeuft es so weiter.`

Die Hypothese ist eine Gedankenspur. Sie beschreibt, was DIO sich bei einer
realen Lage gedacht hat. Sie ist nicht die reale Lage selbst.

## 6. Realitaetspruefung

DIO prueft:

- Passt die Hypothese zur aktuellen sichtbaren Form?
- Passt die Hypothese zur aktuellen MCM-Lage?
- Gibt es reale Erfahrung mit verwandten Formen?
- Hat diese Art von Gedanke in aehnlichen Lagen getragen?

Hier entsteht Reife, Vertrauen, Vorsicht oder Distanz.

## 7. Handlung

Ein Entry wird erst gesucht, wenn tragende Naehe entsteht.

Wichtig:

- Entry-Anker ist reale Form-/Bereichs-/MCM-Naehe.
- Hypothese erzeugt keinen Entry.
- Hypothese darf bestaetigen, vorsichtiger machen oder Beobachtung naheliegen.
- Das einzige harte Gate bleibt das oekonomische Value-Gate.

## 8. Konsequenz

TP, SL, Cancel, Beobachtung und verpasste Handlung werden zurueckgefuehrt in:

- MCM-Feld
- Memory
- Thought-Memory
- Vorsicht
- Vertrauen
- Reorganisation

Damit lernt DIO nicht nur, ob ein Trade profitabel war, sondern ob seine
Wahrnehmung, sein Denken und seine Handlung getragen haben.

## Leitsatz

`Welt -> Sehen -> MCM-Feld -> Memory -> Thought -> Realitaetspruefung -> Handlung -> Konsequenz`

Nicht:

`Thought -> Handlung`
