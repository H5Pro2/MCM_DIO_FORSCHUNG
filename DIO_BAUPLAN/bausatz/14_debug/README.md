# 14 Debug

Debug ist Beobachtung von außen.

## Aufgabe

Debug macht DIO lesbar, ohne die Mechanik zu bestimmen.

## Darf

- Zustände schreiben
- GUI-Daten schreiben
- Forschungsprotokolle schreiben
- Performance messen
- schlank oder tief laufen

## Darf nicht

- Architektur bestimmen
- Handlungslogik verändern
- DIO durch Protokolllast überladen

## Ausgang

```text
DebugRecord
```
