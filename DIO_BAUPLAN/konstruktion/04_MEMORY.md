# 04 Memory

## Aufgabe

Memory speichert reale Erfahrung.

Memory fragt:

```text
Kenne ich diese reale Form/MCM-Lage?
Gab es ähnliche Lagen?
Was war die Konsequenz?
```

## Realitätsanker

Memory ankert an:

- realer Form
- realer MCM-Wirkung
- realer Konsequenz

Memory ankert nicht an einer freien Hypothese.

## Inhalt

Eine Erfahrung enthält:

- Formfamilie
- MCM-Feldlage
- Richtungskontext
- Zeitkontext
- Handlung oder Nicht-Handlung
- Konsequenz
- spätere Tragfähigkeit

## Eigenschaftsprofile

Memory darf reale Erfahrung nicht nur exakt als Kontaktart wiederholen.
Zusaetzlich braucht DIO eine weiche Verallgemeinerung:

```text
Welche Eigenschaften haben getragen?
Welche Eigenschaften haben belastet?
Welche Eigenschaften waren aehnlich, obwohl die Form nicht exakt gleich war?
```

Diese Spur ist kein Hypothesen-Speicher und keine Strategie-Regel.
Sie entsteht aus realen Konsequenzen und bildet Eigenschaftsprofile:

- Form-/Bereichsqualitaet
- MCM-Tragwirkung
- Kontaktnaehe
- Zeit-/Raumzeitpassung
- Reife, Nutzen, Vorsicht
- Ergebnisbelastung oder konstruktive Wirkung

Damit kann DIO eine neue Lage nicht nur fragen:

```text
Kenne ich exakt diesen Kontakt?
```

sondern auch:

```text
Traegt diese neue Lage Eigenschaften, die frueher tragend oder belastend waren?
```

Das ist kein Auswendiglernen. Es ist reale Erfahrungsverallgemeinerung.
Die Wirkung bleibt weich und darf Handlung nur modulieren, nicht erzwingen.

## Reife Vorsicht ist nicht reife Tragfaehigkeit

Der Jahreslauf 2025 hat gezeigt:

```text
Memory kann reifen, aber in Richtung Vorsicht.
Diese Reife darf nicht wie tragende Sicherheit wirken.
```

Darum unterscheidet DIO bei Kontaktprofilen:

- tragende Reife
- vorsichtige Reife
- unreife Naehe
- belastete Wiederkehr

Wenn eine Form-/MCM-Eigenschaft oft wiederkehrt, aber ueberwiegend Belastung
erzeugt, entsteht kein Blocker. Es entsteht ein koerperlicher Rueckzug:

```text
Ich kenne diese Art Kontakt.
Sie ist nicht leer.
Aber mein Feld erinnert Belastung.
Ich werde vorsichtiger, pruefe laenger oder bleibe beobachtend.
```

Das ist eine weiche Rueckkopplung aus realer Konsequenz. Sie darf Entry-Naehe
verringern und Beobachtung erhoehen, aber keine starre Regel bilden.

## Trennung

Es gibt zwei Speicherarten:

- Welt-/Erfahrungs-Memory
- Thought-Memory

Welt-Memory speichert, was real passiert ist. Thought-Memory speichert, was
DIO gedacht hat.

## Schreibvertrag

In das Welt-/Erfahrungs-Memory darf nur geschrieben werden, was einen
Realitätsanker hat:

```text
reale Form
reale MCM-Wirkung
reale Zeitlage
reale Handlung oder reale Nicht-Handlung
reale Konsequenz oder reale Beobachtungsfolge
```

Eine Hypothese darf im Welt-Memory nur als Bezug erscheinen:

```text
Bei dieser realen Form gab es diesen Gedanken.
```

Sie darf dort nicht als Wahrheit gespeichert werden.

## Thought-Memory

Thought-Memory speichert:

```text
Was habe ich gedacht?
Worauf bezog sich der Gedanke real?
Wurde der Gedanke später bestätigt, widerlegt oder reorganisiert?
```

Damit bleibt die Grenze klar:

- Memory = reale Erfahrung
- Thought-Memory = innere Denkspur zu realer Erfahrung

## Abgrenzung Reflexionsspeicher

Ein spaeterer Reflexionsspeicher ist nicht Welt-Memory.

Welt-Memory speichert:

```text
Was real passiert ist.
```

Reflexionsspeicher duerfte spaeter speichern:

```text
Wie DIO diese reale Erfahrung innerlich gelesen hat.
```

Beispiel:

```text
Welt-Memory:
  Familie X hatte realen negativen Kontakt.
  Spaeter hatte Familie X realen positiven Kontakt.

Reflexionsspeicher:
  Diese Familie wurde passiv als reorganisierte Spur gelesen.
```

Der Reflexionsspeicher darf daraus keine Entry-Erlaubnis, kein Verbot und
keine harte Reife ableiten.
